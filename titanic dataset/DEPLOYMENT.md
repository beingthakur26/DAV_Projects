# 🚀 Titanic Predictor: Deployment Guide

This guide explains how to deploy your AI application to the web using **Render** (for the backend) and **Vercel** (for the frontend).

---

## 1. Prerequisites

1.  **GitHub Repository**: Push your entire `titanic dataset` folder to a new GitHub repository.
2.  **Accounts**:
    *   [Render Account](https://render.com/) (linked to GitHub).
    *   [Vercel Account](https://vercel.com/) (linked to GitHub).

---

## 2. Deploying the Backend (Render)

Render will host your Python Flask API.

1.  **Create a New Web Service**:
    *   Log into Render Dashboard.
    *   Click **New** > **Web Service**.
    *   Select your GitHub repository.
2.  **Configure Settings**:
    *   **Name**: `titanic-survival-api` (or your choice).
    *   **Environment**: `Python 3`.
    *   **Root Directory**: `backend` (⚠️ Very Important).
    *   **Region**: Choose the one closest to you.
    *   **Branch**: `main`.
3.  **Build & Start Commands**:
    *   **Build Command**: `pip install -r requirements.txt`.
    *   **Start Command**: `gunicorn app:app`.
4.  **Advanced Settings**:
    *   Make sure to select the **Free** instance type (unless you want to scale).
    *   Click **Create Web Service**.

**Note**: Your API URL will look like `https://titanic-survival-api.onrender.com`. **Copy this URL.**

---

## 3. Connecting Frontend to Backend

Before deploying the frontend, you must update the API URL in your code.

1.  Open `frontend/assets/js/script.js`.
2.  Replace the placeholder in `PRODUCTION_API_URL` with your Render URL:
    ```javascript
    const PRODUCTION_API_URL = 'https://titanic-survival-api.onrender.com';
    ```
3.  **Commit and Push** this change to GitHub.

---

## 4. Deploying the Frontend (Vercel)

Vercel will host your premium web interface.

1.  **Create a New Project**:
    *   Log into Vercel Dashboard.
    *   Click **Add New** > **Project**.
    *   Import your GitHub repository.
2.  **Configure Build Settings**:
    *   **Project Name**: `titanic-predictor-ui`.
    *   **Framework Preset**: `Other`.
    *   **Root Directory**: `frontend` (⚠️ Very Important).
3.  **Deploy**:
    *   Click **Deploy**.
    *   Vercel will build your static site and provide a URL (e.g., `https://titanic-predictor.vercel.app`).

### Troubleshooting
*   **CORS Errors**: The `app.py` already includes `CORS(app)`, which allows requests from anywhere. If you want to be more secure, you can update `CORS(app, resources={r"/*": {"origins": "your-vercel-url.com"}})` in `app.py`.
*   **Startup Time**: Render's free tier "spins down" after inactivity. The first request after a break might take 30-50 seconds to respond.
